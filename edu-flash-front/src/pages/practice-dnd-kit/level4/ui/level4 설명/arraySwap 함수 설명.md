# arraySwap 유틸리티 함수 분석

이 함수는 `dnd-kit`의 `arrayMove`를 대신하여, **두 아이템의 자리만 1:1로 맞바꾸기 위해** 만든 커스텀 유틸리티 함수입니다.

---

## 1. 코드 분석

```tsx
function arraySwap<T extends { id: string }>(
  arr: T[], 
  fromId: string, 
  toId: string
): T[] {
  // ① 불변성 유지: 원본 배열을 복사하여 새로운 배열을 만듭니다.
  const result = [...arr]
  
  // ② 위치 찾기: 바꾸고자 하는 두 아이템의 인덱스(번호)를 찾습니다.
  const fromIndex = result.findIndex((i) => i.id === fromId)
  const toIndex = result.findIndex((i) => i.id === toId)
  
  // ③ 예외 처리: 만약 아이템을 찾지 못했다면 원래 배열을 그대로 돌려줍니다.
  if (fromIndex === -1 || toIndex === -1) return result
  
  // ④ 핵심(Swap): 위에서 배운 구조 분해 할당으로 자리를 맞바꿉니다.
  // (앞의 세미콜론은 안전장치!)
  ;[result[fromIndex], result[toIndex]] = [result[toIndex], result[fromIndex]]
  
  // ⑤ 결과 반환: 자리가 바뀐 새로운 배열을 반환합니다.
  return result
}
```

---

## 2. 주요 포인트 설명

### [Point 1] 제네릭 활용 (`<T extends { id: string }>`)
이 함수는 어떤 종류의 데이터(문자열, 객체 등)가 들어오든 상관없이 작동하도록 설계되었습니다. 단, 아이템들을 구별하기 위해 최소한 `id`라는 이름의 문자열 속성은 반드시 가지고 있어야 한다는 제약 조건을 걸어둔 것입니다. (타입 안정성 확보)

### [Point 2] 왜 복사(`[...arr]`)를 하나요?
React에서는 원본 데이터를 직접 수정(Mutation)하면 변경 사항을 감지하지 못해 화면이 바뀌지 않을 수 있습니다. 따라서 항상 복사본을 만들어 수정한 뒤, "여기 새로운 데이터가 있어!"라고 알려주는 것이 React의 정석입니다.

### [Point 3] `arrayMove`와의 차이점
- **`arrayMove`**: 한 친구를 뽑아서 다른 친구들 사이로 억지로 밀어 넣습니다. (나머지 친구들이 밀려남)
- **`arraySwap`**: 복도에서 마주친 두 친구가 서로의 명찰(자리)만 딱 바꾸고 지나가는 것과 같습니다. (다른 친구들은 아무 영향 없음)

## 3. 사용 예시

```tsx
// 결과: [{id: 'C'}, {id: 'B'}, {id: 'A'}]
// B는 자리를 지키고 A와 C만 바뀌었습니다!
```

## 🔎 보충 설명: `function arraySwap<T extends { id: string }>` 타입이 왜 이렇게 요란한가요?

이 한 줄은 **"범용성"**과 **"안전성"**을 동시에 잡기 위한 타입스크립트의 고난도 문법입니다.

### 1. `<T>` : "무엇이든 다 받겠다" (범용성)
`T`는 특정 타입이 아니라 "어떤 타입이 들어올지 모른다"는 뜻의 플레이스홀더입니다. 덕분에 이 함수는 카드 리스트뿐만 아니라 학생 명단, 상품 목록 등 어떤 배열이든 처리할 수 있는 **'무적의 범용 유틸리티'**가 됩니다.

### 2. `extends { id: string }` : "단, ID는 꼭 있어야 한다" (안정성)
함수 내부에서 `i.id`를 꺼내 쓰고 있기 때문에, `id`가 없는 데이터가 들어오면 에러가 납니다. 
- `extends`를 통해 "들어오는 모든 타입(`T`)은 최소한 `id`라는 문자열은 꼭 가지고 있어야 해!"라고 **입구 컷**을 하는 것입니다.

### 3. 결론
이 요란한 설정 덕분에, 우리는 **"id만 있다면 그게 무엇이든 안전하게 자리를 바꿔주는 무적의 함수"**를 얻게 된 것입니다. 😊
