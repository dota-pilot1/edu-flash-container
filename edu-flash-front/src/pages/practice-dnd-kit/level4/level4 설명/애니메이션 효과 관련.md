# Level 4: 애니메이션 효과 분석 — 시각적 핵심 포인트

## 1. 코드 예제 (GridCell 컴포넌트)

```tsx
return (
  <motion.div
    layout                                       // 효과 2의 핵심
    transition={{ type: 'spring', stiffness: 400, damping: 28 }}
    ref={(node) => { setDragRef(node); setDropRef(node) }}
    {...attributes}
    {...listeners}
    className={`flex h-24 cursor-grab items-center justify-center rounded-lg border text-2xl font-bold shadow-sm select-none ${item.color} ${
      isDragging
        ? 'opacity-0'
        : isOver && activeId !== item.id
          ? 'ring-2 ring-blue-400 scale-110'    // 효과 1의 핵심
          : 'hover:shadow-md'
    }`}
  >
    {item.label}
  </motion.div>
)
```

---

## 2. 질문 (Question)

**아이템을 드래그하여 다른 아이템 위에 올리고(Over), 내려놓았을 때(End) 각각 어떤 애니메이션 효과가 발생하나요?**

---

## 3. 분석 (Analysis with Code)

### [애니메이션 효과 1] 드래그 오버(Drag Over) 시: 해당 번호가 커지는 효과

사용자가 아이템을 잡고 다른 타겟 아이템 위에 마우스를 올렸을 때 발생하는 즉각적인 시각 피드백입니다.

- **분석**: 마우스가 올라간 아이템이 드롭 가능한 타겟임을 사용자에게 명확히 알려줍니다.
- **코드 설명**: 
  ```tsx
  isOver && activeId !== item.id ? 'ring-2 ring-blue-400 scale-110' : ''
  ```
  `isOver` 상태(마우스가 위에 있음)가 되면 `scale-110` 클래스가 적용되어 **아이템이 1.1배 커지고**, 푸른색 테두리가 생기며 강조됩니다.

---

### [애니메이션 효과 2] 드래그 앤 드롭(Drag End) 시: 부드럽게 교체 되는 효과

아이템을 내려놓는 순간, 두 아이템이 순간이동하지 않고 부드럽게 미끄러지며 자리를 교체하는 효과입니다.

- **분석**: 데이터 순서가 바뀌었을 때 화면이 뚝뚝 끊기지 않고 물리적으로 연결된 느낌(UX)을 줍니다.
- **코드 설명**: 
  ```tsx
  <motion.div layout transition={{ type: 'spring', ... }}>
  ```
  1. **`layout`**: 데이터(배열 순서)가 바뀌었을 때, `framer-motion`이 아이템의 새 위치를 감지하여 부드러운 이동 경로를 자동으로 생성합니다.
  2. **`transition(spring)`**: 스프링 물리 엔진을 적용하여, 아이템이 제자리에 도착할 때 자석처럼 쫀득하게 정착하는 느낌을 더해줍니다.

---

## 💡 요약 결론

1. **오버(Over)** 시에는 **강조(`scale`)**를 통해 사용자에게 "여기가 정답이야!"라고 알려주고,
2. **드롭(End)** 시에는 **이동(`layout`)**을 통해 "부드럽게 자리가 바뀌었어!"라고 최종 확인을 시켜주는 구조입니다. 😊
