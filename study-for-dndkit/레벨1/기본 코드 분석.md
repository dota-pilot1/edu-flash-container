# Level 1 — 기본 드래그 앤 드롭 분석

레벨 1 단계에서 사용된 핵심 요소들과 구현 단계를 정리합니다.

## 1. 핵심 구성 요소

이 예제에서 드래그 앤 드롭을 구현하기 위해 사용된 5가지 핵심 요소입니다.

- **`DndContext`**: 드래그 앤 드롭 기능을 제공하는 전체 컨텍스트입니다. 모든 드래그/드롭 컴포넌트는 이 안에 위치해야 합니다.
- **`useDraggable`**: 특정 컴포넌트를 "드래그 가능"하게 만드는 훅입니다. (`DraggableItem`에서 사용)
- **`useDroppable`**: 특정 영역을 "드롭 가능"하게 만드는 훅입니다. (`DroppableZone`에서 사용)
- **`handleDragEnd`**: 드래그가 끝났을 때(아이템을 놓았을 때) 호출되어 사후 처리를 담당하는 함수입니다.
- **`dropped`**: 드래그 아이템이 어느 영역에 놓였는지에 대한 정보를 담는 상태(`state`)입니다.

---

## 2. 구현 단계 및 코드 분석

실제 코드에서 로직이 실행되는 순서(Step 1~3)를 기준으로 분석합니다.

### [Step 1] 상태 선언
먼저, 아이템이 어디에 드롭되었는지를 추적하기 위한 상태를 선언합니다.
```tsx
// dropped: 드롭된 장소의 ID를 저장 (초기값 null)
const [dropped, setDropped] = useState<string | null>(null)
```

### [Step 2] 위치 감지 (over)
드래그가 끝나는 시점에 어떤 영역 위에서 끝났는지 확인합니다.
```tsx
const handleDragEnd = (event: DragEndEvent) => {
  // 이벤트가 일어난 장소(드롭 영역) 정보를 over 변수로 받습니다.
  const { over } = event
```

### [Step 3] 상태 업데이트 및 성공 처리
만약 유효한 드롭 영역(`over`)이 존재한다면, 해당 영역의 `id`를 상태에 저장합니다.
```tsx
  // over가 있으면(즉, 드롭 영역 위에서 놓았다면)
  if (over) {
    // 장소의 id(예: 'zone-a', 'zone-b')로 상태를 업데이트합니다.
    setDropped(over.id as string)
  }
}
```
> **참고**: 이때 사용되는 `over.id`는 `DroppableZone` 컴포넌트의 props로 설정된 `id` 값을 참조하게 됩니다.

---

## 3. 요약 (구조적 이해)
1. **`DndContext`**로 감싼다.
2. **`useDraggable`**로 아이템을 만든다.
3. **`useDroppable`**로 영역을 만든다.
4. **`handleDragEnd`**에서 `over` 정보를 확인한다.
5. **`dropped`** 상태를 업데이트하여 화면에 성공 결과를 보여준다.
