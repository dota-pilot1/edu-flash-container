# Step 8 — 통합 테스트

## 환경 준비
1. Docker Compose → PostgreSQL 실행
2. Spring Boot 서버 실행 (localhost:8080)
3. Vite dev 서버 실행 (localhost:5173 또는 5174)

## 테스트 시나리오

### 8-1. 초기 상태
- [ ] 헤더에 1차 메뉴 4개 (홈, 학습, 내 카드, 설정) 표시
- [ ] 하위 메뉴 없는 메뉴 클릭 → 페이지 이동
- [ ] 브라우저 콘솔 에러 없음

### 8-2. 메뉴 관리 CRUD
- [ ] /menu-manage 접속 → 트리 + 폼 레이아웃
- [ ] 1차 메뉴 인라인 추가 (하단 버튼)
- [ ] 2차 메뉴 인라인 추가 (우클릭 → 하위 메뉴 추가)
- [ ] 3차 메뉴 추가 (우클릭)
- [ ] 3차 메뉴에서 "하위 메뉴 추가" 비활성화 확인
- [ ] 메뉴 수정 (이름/경로/순서/노출) → 저장
- [ ] 메뉴 삭제 → 하위 메뉴 cascade 삭제

### 8-3. 헤더 드롭다운
- [ ] 하위 메뉴 추가 후 → 해당 1차 메뉴에 드롭다운 화살표 표시
- [ ] 드롭다운 클릭 → 하위 메뉴 목록 표시
- [ ] 하위 메뉴 클릭 → 페이지 이동
- [ ] 바깥 클릭 → 드롭다운 닫기

### 8-4. 엣지 케이스
- [ ] 중복 path 입력 시 에러 처리
- [ ] visible OFF → 헤더에서 숨김, 트리에서는 투명도
- [ ] Vite 포트 5174에서도 API 정상 동작 (프록시)
