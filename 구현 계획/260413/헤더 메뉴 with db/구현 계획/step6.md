# Step 6 — 프론트: 헤더 DB 메뉴 + 드롭다운

## 목표
헤더에 DB 1차 메뉴 동적 렌더링, 하위 메뉴 있으면 드롭다운

## 작업 내용

### 6-1. Header 컴포넌트 수정
- `useQuery(['menus', 'tree'])` → 트리 데이터로 children 정보 확보
- visible=true 메뉴만 표시

### 6-2. HeaderMenu 서브 컴포넌트
| 조건 | 클릭 동작 |
|------|----------|
| children 없음 | `<Link>` → 바로 페이지 이동 |
| children 있음 | `<button>` → 드롭다운 토글 |

### 6-3. 드롭다운 UI
- absolute 포지션, 메뉴 아래 표시
- 하위 메뉴 목록 → 클릭 시 navigate + 닫기
- 현재 경로 active 표시 (하위 메뉴 포함)
- 바깥 클릭 → 닫기
- 화살표 아이콘 (▼) 표시

### 6-4. "메뉴 관리" 링크
- 하드코딩으로 구분자(|) 뒤에 고정 배치
- 관리자용 링크

## 완료 기준
- [ ] 하위 메뉴 없는 1차 메뉴 → 클릭 시 바로 이동
- [ ] 하위 메뉴 있는 1차 메뉴 → 클릭 시 드롭다운
- [ ] 드롭다운에서 하위 메뉴 클릭 → 페이지 이동
- [ ] DB 메뉴 변경 시 헤더 자동 반영
